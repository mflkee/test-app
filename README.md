# Итоговое задание по тестированию

Репозиторий содержит результаты ручного и автоматизированного тестирования учебного сервиса денежных переводов `F-Bank`.

## Состав репозитория

- `manual-tests/` — 5 тест-кейсов для ручной проверки
- `bug-reports/` — баг-репорты по найденным дефектам
- `tests/` — Selenium UI-тесты на `pytest`
- `.github/workflows/selenium-tests.yml` — конфигурация GitHub Actions
- `.github/ISSUE_TEMPLATE/` — шаблон для оформления багов в GitHub Issues
- `dist/` — выданная сборка сервиса

## Найденные дефекты

1. `BUG-01` — поле номера карты принимает 17 цифр вместо 16
2. `BUG-02` — комиссия рассчитывается неверно для сумм, не кратных 10
3. `BUG-03` — перевод блокируется при точном равенстве суммы с комиссией доступному остатку

## Локальный запуск сервиса

По инструкции сервис можно запустить так:

```bash
python3 -m http.server 8000 --directory dist
```

После запуска открыть:

```text
http://localhost:8000/?balance=30000&reserved=20001
```

## Запуск автотестов

Автотесты сами поднимают локальный HTTP-сервер для каталога `dist`, поэтому отдельно запускать сервис перед `pytest` не нужно.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pytest -vv
```

## Почему CI красный

Сборка намеренно остаётся красной: автотесты описывают корректное ожидаемое поведение для открытых дефектов `BUG-01`, `BUG-02` и `BUG-03`, поэтому на текущей версии сервиса они падают. Это соответствует требованиям задания.

## GitHub Issues

Для новых багов в репозитории подготовлен шаблон issue. Он повторяет структуру файлов из `bug-reports/`, чтобы дефекты в GitHub оформлялись в том же формате.
